Deployment entry point: start.mjs — uses direct import (NOT child process spawn):

5. DARKWAVE ECOSYSTEM HUB
Server file: server/ecosystemHub.ts
Route registration: server/routes.ts (lines ~10565-11000)

Three classes:

DarkWaveEcosystemHub — Server-side hub that receives connections
EcosystemClient — Client class for connecting TO other hubs
ExternalHubManager — Manages ORBIT's outbound hub connections
Database tables (Drizzle schema in shared/schema.ts starting line ~5786):

Table	Purpose
ecosystem_connected_apps	Registered apps with API keys, permissions, metadata
ecosystem_code_snippets	Shared code between apps
ecosystem_data_syncs	Sync records (workers, contractors, timesheets, etc.)
ecosystem_activity_logs	Activity audit trail
ecosystem_external_hubs	Outbound hub connections
ecosystem_financial_transactions	Cross-app payment sync
API Endpoints:

Method	Endpoint	Auth	Purpose

GET	/api/ecosystem/snippets	ecosystemAuth	Get code snippets

POST	/api/ecosystem/sync/contractors	ecosystemAuth	Sync contractors

POST	/api/ecosystem/sync/certifications	ecosystemAuth	Sync certifications
GET/POST	/api/ecosystem/logs	ecosystemAuth	Activity logs
Authentication middleware (ecosystemAuth):

Reads X-API-Key and X-API-Secret from request headers
Looks up app in ecosystem_connected_apps table
Verifies secret via SHA-256 hash comparison
Sets (req as any).ecosystemApp = app
API key format: dw_app_ + 24 random hex bytes
API secret format: dw_secret_ + 32 random hex bytes

Permission system: Checks app.permissions array. sync:all grants full access. Individual permissions: read:code, write:code, read:workers, write:workers, read:logs, write:logs, write:1099, write:contractors, write:timesheets, write:certifications.
